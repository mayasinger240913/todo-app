# 🚀 איך מעלים את "טודו" לאינטרנט (מדריך למתחילים)

המטרה: כתובת **https קבועה** שעובדת תמיד (גם כשהמחשב כבוי), שאפשר להפיץ.
נשתמש ב-**Render** — שירות אחסון עם שכבה חינמית ו-https אוטומטי.

> זמן משוער: 20–30 דקות. צריך שני חשבונות חינמיים: GitHub ו-Render.

---

## שלב 1: להעלות את הקוד ל-GitHub

GitHub הוא "ענן" לקוד. Render מתחבר אליו ומפרסם את האפליקציה.

1. פותחים חשבון חינם ב-https://github.com
2. יוצרים מאגר (Repository) חדש בשם למשל `todo-app` (אפשר Private).
3. במחשב, בתוך תיקיית `chore-app`, מריצים ב-PowerShell:
   ```powershell
   cd "chore-app"
   git init
   git add .
   git commit -m "טודו - גרסה ראשונה"
   git branch -M main
   git remote add origin https://github.com/USERNAME/todo-app.git
   git push -u origin main
   ```
   (מחליפים `USERNAME` בשם המשתמש שלך ב-GitHub.)

> ✅ קובץ `.gitignore` כבר דואג שלא יעלו נתוני משתמשים (chores.db, uploads).

---

## שלב 2: לפרסם ב-Render

1. נרשמים חינם ב-https://render.com (אפשר להתחבר עם חשבון ה-GitHub).
2. לוחצים **New +** → **Web Service**.
3. מחברים את מאגר ה-GitHub שיצרת (`todo-app`).
4. Render יזהה את הקובץ `render.yaml` ויגדיר הכל לבד. אם מבקש ידנית:
   - **Build Command:** `pip install -r requirements.txt`
   - **Start Command:** `gunicorn app:app`
   - **Plan:** Free
5. לוחצים **Create Web Service** וממתינים כמה דקות.
6. מקבלים כתובת קבועה כמו `https://todo-app.onrender.com` — **זהו! זה באוויר** 🎉

---

## שלב 3: אחרי הפרסום

- פותחים את הכתובת בטלפון → המצלמה עובדת (כי זה https) → "הוסף למסך הבית".
- מעדכנים את כתובת מדיניות הפרטיות (הקישור `/privacy`) בחנויות/בעמוד השיווק.

---

## ⚠️ דברים שחשוב לדעת על השכבה החינמית

1. **"שינה":** אחרי ~15 דקות בלי שימוש השרת נרדם, והכניסה הראשונה אחריה
   לוקחת ~30 שניות לעלות. (בתוכנית בתשלום זה נעלם.)

2. **הנתונים נמחקים בכל פרסום מחדש!** השכבה החינמית לא שומרת קבצים לאורך זמן,
   אז `chores.db` והתמונות יימחקו בכל deploy או הפעלה מחדש. זה **בסדר לבדיקות**,
   אבל **לא למוצר אמיתי**. לפני השקה אמיתית צריך:
   - **Persistent Disk** ב-Render (בתשלום), או
   - מסד נתונים אמיתי כמו **PostgreSQL** (ל-Render יש חינמי לתקופה), או
   - אחסון תמונות בשירות ענן (כמו S3 / Cloudinary).
   זה שלב הנדסי נוסף שכדאי לעשות כשעוברים מ"בדיקה" ל"מוצר".

3. **מפתח סודי:** Render ייצר אוטומטית `SECRET_KEY` (מוגדר ב-render.yaml).
